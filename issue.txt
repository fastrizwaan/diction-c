Worked for 4m
Done. I replaced the RAM FTS index with a disk-backed SQLite FTS5 trigram index.

Key changes:

Removed the in-memory trigram posting-list index.
Added persistent SQLite FTS indexes under cache: ~/.cache/diction/fts/*.sqlite.
Index is not created by default.
Added a Preferences switch: System -> Search -> Enable Full Text Search Index.
Full Text Search mode is blocked unless that setting is enabled.
No full-scan fallback in FTS mode, so it won’t secretly burn CPU/RAM if the disk index has no hit.
SQLite is configured with a tiny cache and file-backed temp storage.
Indexing commits in small batches with tiny pauses to reduce disk/CPU pressure.
Search behavior:

First FTS query for a dictionary builds its disk index.
Later FTS queries use SQLite FTS5 trigram search, then verify the exact existing regex match.
Queries shorter than 3 chars return no FTS result instead of doing a painful scan.

./build.sh

It passes. Existing unrelated warnings remain for unused functions.
